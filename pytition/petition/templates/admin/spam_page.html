{% trans "Cron commands" %}

{% blocktrans %}

Petitions require their signatures to be checked periodically to avoid spam. You can change this and make the verification at each signatures by using the switch check signatures at each signature action in petitions tables.
Permissions for organizations are also checked periodically. If a user is moderated, they lose their permissions in their organizations.

{% endblocktrans %}

{% trans "In order to launch these Cron jobs, you can put the following commands in crontab by using the crontab -e command at the root of the project, outside of the docker container:" %}

  • {time} docker exec {docker_container_name} /venv/.venv/bin/python3.8 pytition/manage.py cron
  • {time} docker exec {docker_container_name} /venv/.venv/bin/python3.8 pytition/manage.py cron_permissions

{% trans "Tests to avoid inappropriate petitions" %}

{% trans "In order to avoid inappropriate petitions, we use detectors defined in pytition/petition/spam_management/detectors. They are all called in pytition/petition/spam_management/detector.py." %}

  • {% trans "akismet_detector: uses Akismet's is_spam() function to detect spam content. Returns 0, 1 or 2." %}
  • {% trans "alt_profanity_check_detector: uses function predict_prob() from alt-profanity-check library to detect profanity in english content. Returns 0, 1 or 2." %}
  • {% trans "keywords_detector: uses list of forbidden keywords defined in settings. Returns 0 or 1." %}

{% trans "Depending on the results of these detectors, automatic moderation or monitoring actions are taken:" %}

  • {% trans "0: no spam, no moderation or monitoring." %}
  • {% trans "1: possible spam, strong monitoring." %}
  • {% trans "2: spam, moderation." %}

{% trans "If there are too many false positives with Akismet, you can disable automatic moderation by putting AKISMET_MODERATION_AUTO at False (pytition/pytition/settings/base.py). That way, Akismet results will only trigger monitoring." %}

{% trans "Monitoring variables" %}

{% trans "The variables defined as VARIABLE_PRIORITY trigger monitoring with said priority. Four types of monitoring are defined:" %}

Critical: the petition is moderated, emails are sent to admin and owner.

Strong: the petition is monitored with a strong priority, emails are sent to admin and user, display in admin and owner dashboard.

Average: the petition is monitored with an average priority, display in admin and owner dashboard.

Low: the petition is monitored with a low priority, display in admin.
